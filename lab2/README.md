# 🌐 Lab 2: IPv6 Addressing, ICMPv6, and Security

## 📖 Overview
This module explores the IPv6 protocol suite, its automatic configuration mechanisms, and inherent security vulnerabilities.  
The exercises progress from manual IPv6 assignment to automated deployments using SLAAC and DHCPv6 Prefix Delegation (PD). It concludes with practical experiments on Path MTU discovery, IPv4 transition tunnels, and executing real-world Neighbor Discovery Protocol (NDP) attacks.

---

## 🛠️ Exercises

### [Exercise 1](./ex1/): Manual IPv6 Configuration
**Objective:** Statically assign IPv6 addresses and verify routing capabilities.

* **Task:** Manually configure `pc1`, `pc2`, `pc3`, and `pc4` to reside across two different IPv6 subnetworks, with router `r1` acting as their default gateway.
* **Constraints:** 
  * Configure `pc1` and `pc4` using the Debian `/etc/network/interfaces` file.
  * Configure `pc2` using the `ip` command suite.
  * Configure `pc3` using the legacy `ifconfig` command.
  * Set the default gateway to `r1` using its link-local address.
  * Verify connectivity using `ping`, observing the difference between pinging a link-local address versus a Global Unicast Address (GUA).


### [Exercise 2](./ex2/): SLAAC Configuration
**Objective:** Automate IPv6 network configuration using Stateless Address Autoconfiguration.

* **Task:** Configure the four PCs to automatically receive their IPv6 networking configuration from the router using default SLAAC mechanisms.
* **Constraints:** 
  * Modify Linux `sysctl` parameters to enable proper autoconfiguration.
  * The Router Advertisement Daemon (`radvd`) must be started manually on the router.

> **🔍 Traffic Analysis**
> * The task requires using `tcpdump` to capture the ICMPv6 Router Advertisement (RA) and Router Solicitation (RS) messages exchanged on the network.
> * Captures should demonstrate how the PCs generate their Interface IDs (e.g., EUI-64 or privacy extensions).


### [Exercise 3](./ex3/): SLAAC + Stateless DHCPv6
**Objective:** Combine SLAAC for routing configuration with stateless DHCPv6 for secondary network parameters.

* **Task:** Configure the router and PCs so that hosts receive their IPv6 prefix via SLAAC, but request their DNS server information via stateless DHCPv6.
* **Constraints:** 
  * The router must correctly advertise the prefix, route, and the stateless directive (O flag = 1) using `radvd` (via `sysctl`) and `dnsmasq`.
  * Start `dnsmasq` in the foreground (`-d` option) to easily monitor its operation.

> **🔍 Traffic Analysis**
> * The task requires capturing the Router Advertisements and the subsequent DHCPv6 Information-Request and Reply exchanges.


### [Exercise 4](./ex4/): DHCPv6 Prefix Delegation (PD)
**Objective:** Implement dynamic subnet allocation across a multi-router topology.

* **Task:** Configure an internal router to request an IPv6 prefix from an ISP router (which acts as the DHCPv6 server) and distribute `/64` subnets inside two internal LANs (2 PCs each).
* **Constraints:** 
  * The ISP is already pre-configured to provide prefixes.
  * The internal router must be configured using either `dibbler` (DHCPv6 client) + `radvd` OR `wide-dhcp` + `dnsmasq`.
  * The router must distribute the assigned addresses to the internal LANs using SLAAC.


### [Exercise 5](./ex5/): ICMPv6 MTU Discovery
**Objective:** Experiment with Path MTU discovery and network fragmentation handling in IPv6.

* **Task:** Manage a topology of three routers connecting two LANs. Manually alter the Maximum Transmission Unit (MTU) on the links between the routers to force packet drops.
* **Constraints:** 
  * Use the command `ip link set mtu XXXX dev YYY` on both endpoints of a link to alter the MTU (e.g., setting it to 1480 or 1450).
  * Use `ping` and `tracepath` to test connectivity and force MTU discovery across the bottleneck links.

> **🔍 Traffic Analysis**
> * Use `tcpdump` to capture the resulting ICMPv6 "Packet Too Big" messages generated by the routers when a packet exceeds the link's MTU.


### [Exercise 6](./ex6/): IPv6 Tunneling
**Objective:** Create an IPv6-capable connection over an IPv4-only infrastructure.

* **Task:** Establish a virtual tunneling interface using Intra-Site Automatic Tunnel Addressing Protocol (ISATAP) to route IPv6 traffic inside IPv4 headers.
* **Constraints:** 
  * Utilize services from Hurricane Electric (tunnelbroker.net) or manually configure the endpoints.
  * Configure the tunnel using `ip tunnel add mode sit` and assign the appropriate IPv6 routing prefixes to the virtual interface.


### [Exercise 7](./ex7/): IPv6 Security Exercises
**Objective:** Replicate the RIPE NCC IPv6 Security Training labs to execute network reconnaissance and Neighbor Discovery Protocol (NDP) spoofing attacks.

* **Task:** Utilize attack frameworks to map the local IPv6 network, poison neighbor caches, and inject rogue network configurations.
* **Constraints:** 
  * Use `Scapy` (interactive Python packet manipulation) on `pc3`.
  * Compile and install **THC-IPV6** and the **IPv6 Toolkit** on `pc1` and `pc2`.
  * **Network Scanning:** Use tools like `alive6` and `scan6` to discover active hosts.
  * **NDP Threats (NS/NA):** Craft malicious ICMPv6 Neighbor Solicitation and Advertisement messages to execute Man-in-the-Middle (MITM) or Denial of Service (DoS) attacks.
  * **Rogue RA:** Inject unauthorized Router Advertisements to manipulate target host auto-configuration and routing tables.

> **🔍 Security Testing & Evidence**
> * The execution requires collecting `.pcap` files and screenshots as proof of successful cache poisoning and unauthorized route injection.
> * *Note: Exercises related to Multicast Listener Discovery (MLD) scanning and iptables traffic filtering are documented in the RIPE manual but may be excluded based on specific assignment directives.*
