  # 🌐 Lab 1: Networking 101 and Traffic Monitoring

## 📖 Overview
This module introduces the fundamentals of IPv4 network configuration, routing, and packet analysis. The exercises progress from manual interface configurations to setting up dynamic addressing, building static routes across multiple subnets, and ultimately using packet sniffers to dissect network traffic.

---

## 🛠️ Exercises

### Exercise 1: Manual Network Configuration
**Objective:** Statically assign IP addresses and verify connectivity.

* **Task:** Manually configure `pc1`, `pc2`, and `pc3` to be on the same network as the `r1` router (`192.168.100.30/29`).
* **Constraints:** * Configure `pc1` using the Debian `/etc/network/interfaces` file.
  * Configure `pc2` using the `ip` command suite.
  * Configure `pc3` using the legacy `ifconfig` command.
  * Set the default gateway to `r1`, configure DNS, and verify outbound Internet connectivity.

> **🔍 Traffic Analysis (Part of Exercise 4)**
> * The task requires capturing basic ICMP and ARP traffic from this manually configured network using `tcpdump`.
> * The resulting `.pcap` files can be found in the `captures/` directory.


### Exercise 2: DHCP Configuration
**Objective:** Automate network configuration and analyze the DHCP protocol.

* **Task:** Configure `r1` to act as a DHCP server for the `192.168.100.16/28` address pool.
* **Constraints:** * Configure `pc1` and `pc2` to automatically request their IP, DNS, and gateway settings from the DHCP server.

> **🔍 Traffic Analysis (Part of Exercise 4)**
> * The task requires capturing the D.O.R.A. handshake (Discover, Offer, Request, Acknowledge) generated by the clients.
> * The resulting `.pcap` files, which demonstrate how the DHCP server distributes the subnet mask and DNS options, can be found in the `captures/` directory.


### Exercise 3: Subnetting & Static Routing
**Objective:** Design a subnetting scheme and implement static routing across multiple networks.

* **Task:** Given the `172.16.0.0/16` network, assign appropriate subnets to a topology featuring 2 LANs (with 2 PCs each) and an internal transit network.
* **Constraints:** * `r0` acts as the default gateway to the Internet.
  * `r1` and `r2` act as the default gateways for LAN1 and LAN2, respectively.
  * Configure static routes (`ip route`) on all routers to ensure both LANs can reach each other and the Internet.

> **🔍 Traffic Analysis (Part of Exercise 4)**
> * The task requires capturing traffic between the two LANs from multiple vantage points (LAN1, LAN2, and the internal transit network) to observe how MAC addresses change at each router hop.
> * The resulting `.pcap` files can be found in the `captures/` directory.


### Exercise 4: Web & FTP Traffic Monitoring
**Objective:** Use packet sniffing to capture and extract sensitive information from unencrypted application-layer traffic.

* **Task:** The core assignment is to deploy `tcpdump` to capture network traffic while interacting with external and internal unencrypted services.
* **Execution:** 
  * Traffic was generated by browsing to the local webserver (`ba.php` and `da.php`) and authenticating using the provided credentials (`user=angelo`, `psw=angsp`).
  * Additional traffic was generated by connecting to an external FTP server.
  * The resulting `.pcap` files, which can be analyzed in Wireshark to locate the transmitted plaintext credentials, are stored in the `captures/` directory.
